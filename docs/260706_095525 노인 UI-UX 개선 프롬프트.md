# 노인 사용자를 위한 UI/UX 개선 프롬프트

## 배경

`C:\Dev\EPITECH\Midterm\memory-afterimage` 는 "기억의 잔상(Memory Afterimage)" — 노인 사용자가 여행 기억을 음성으로 말하면 AI가 나레이션·이미지·자막을 합성해 하나의 영상으로 만들어주는 React 앱이다.

주요 화면 흐름: `/input` (음성/텍스트 입력) → `/chat` (AI와 대화하며 기억 다듬기) → `/process` (영상 생성 파이프라인) → 결과 영상 확인/다운로드. 그 외 `/new` (세션 설정), `/memories` (보관함), `/settings` (API 키·모델 설정).

핵심 파일: `src/App.tsx`, `src/components/{VoiceInput,VoiceChat,Chat,Pipeline,NewSession,Memories,Settings,StageCard,AdvancedSettings}.tsx`, 스타일은 Tailwind 기반.

## 목표

이 프로젝트의 UI/UX를 **노인(고령 사용자)** 이 무리 없이 쓸 수 있도록 개선하라. 특히 시력 저하, 소근육 조작 어려움, 디지털 기기 낮은 친숙도, 인지 부하 민감성을 고려해야 한다.

## 개선 요구사항 체크리스트

### 1. 가독성
- 본문/버튼 텍스트 최소 크기 상향 (기존 11~14px 수준 `mini-label`, `text-[13px]` 등 → 16px 이상 권장)
- 명도 대비(contrast) WCAG AA 이상 확보. 특히 `text-text-dim` 같은 흐린 회색 텍스트 재검토
- 자막 배경/글자색 커스터마이징 기능(`captionBgColor`, `captionTextColor`)은 유지하되 기본값을 고대비로 조정

### 2. 터치/클릭 타겟
- 버튼·아이콘 최소 44x44px 이상 확보 (`fullscreen-image-button`, nav 버튼 등 점검)
- 버튼 간 간격 충분히 확보해 오터치 방지

### 3. 인지 부하 축소
- 한 화면에 한 가지 행동만 유도 (현재 `/process` 화면처럼 여러 카드 동시 노출 지양하거나 단계별 강조)
- 진행 상태를 항상 명확히 표시 (현재 `pipelineState`/`composeProgress` 기반 로딩 UI 유지·강화)

### 4. 음성 우선 흐름 보강
- `VoiceInput`이 기본 입력 수단이므로, 녹음 시작/종료/재시도를 아주 크고 명확한 버튼과 아이콘+텍스트 라벨로 제공
- 마이크 인식 실패 시 재시도 안내를 부드럽고 반복 가능하게 (겁주는 에러 문구 지양)
- 텍스트 입력 대체 수단(`inputMode 'text'`)도 큰 폰트의 textarea 유지

### 5. 네비게이션 단순화
- 상단 nav(기억 남기기/보관함/설정) 라벨을 그대로 유지하되 아이콘 병기 고려
- "처음부터 진행하기" 같은 되돌리기 버튼의 위치를 일관되게, 실수로 누르지 않도록 확인 단계(confirm) 추가 검토

### 6. 피드백/오류 처리
- `alert()` 네이티브 팝업 대신 앱 톤에 맞는 큰 모달/배너로 교체하고 문구를 쉬운 말로 순화
- 성공/완료 시에도 명확한 시각+문구 피드백 (현재 "당신의 기억이 완성되었습니다" 톤 유지)

### 7. 설정 화면 (`Settings.tsx`, `AdvancedSettings.tsx`)
- API 키 등 기술적 설정은 고급 설정으로 접혀 있는 현재 구조 유지 (최근 커밋에서 이미 분리됨)

## 진행 방식 지침

- 기존 스타일/구조를 유지하며 점진적으로 개선 (전면 재작성 금지)
- 접근성 속성(`aria-label`, `aria-live` 등)은 이미 다수 사용 중 — 계속 일관되게 유지
- 하드코딩된 색상 대비 값은 임의로 바꾸지 말고, 실제 대비비를 계산해 근거를 남길 것
